- **Tailwind CSS v4 + Theme Architecture (2026-03-13T01:45:00Z):** Installed `tailwindcss` + `@tailwindcss/vite` in deck-engine. Vite plugin (`vite.js`) now exports `deckPlugin()`, `deckPlugins()` (combo), and `tailwindPlugin()`. Created `themes/` directory with pluggable theme system: `dark.css`, `light.css`, `shadcn.css` — each has CSS custom properties + `@theme inline` bridge for Tailwind utility generation. Theme loader (`themes/theme-loader.js`) provides `resolveTheme()` and `getAvailableThemes()`. New themes = drop a CSS file. Generated projects include `tailwindcss` + `@tailwindcss/vite` in devDeps. Backwards compatible — existing decks without `theme` field still work. 49 tests passing (was 33). THEME-001 filed.
- **Bug Fix: tailwindPlugin export + CLI polish (2026-03-14):** Root cause of `tailwindPlugin` SyntaxError: published npm v1.8.2 predated the theme work, so downstream projects got a `vite.js` without `tailwindPlugin`. Fixes: (1) Bumped engine version `1.8.2` → `1.9.0` to reflect new tailwind/theme exports. (2) Added `"./vite.js": "./vite.js"` to exports map so both `@deckio/deck-engine/vite` and `@deckio/deck-engine/vite.js` resolve correctly. (3) Updated scaffolder dep `^1.8.2` → `^1.9.0`. CLI UX overhaul: ANSI-colored prompts (`fmt` helpers), default accent changed `#3fb950` → `#6366f1` (indigo-500, more modern), accent color swatch preview via true-color ANSI, theme picker shows descriptions (deep space / projection / editorial neutral), styled success output. No external deps added (no gum). 94 tests passing.
- **Critical Bug Fix: Production build "dirname" error (2026-03-14):** `index.js` re-exported `getAvailableThemes`, `resolveTheme`, `DEFAULT_THEME`, `BUILTIN_THEMES` from `themes/theme-loader.js`, which imports Node.js builtins (`fs`, `path`, `url`). Rollup can't resolve these browser-side during production builds — dev server hid the issue because Vite doesn't bundle in dev mode. Fix: (1) Removed theme-loader re-exports from `index.js` — browser entry now exports only React components, hooks, and constants. (2) Added re-exports of all theme-loader utilities (`resolveTheme`, `getAvailableThemes`, `DEFAULT_THEME`, `BUILTIN_THEMES`) from `vite.js` so Node.js consumers access them via `@deckio/deck-engine/vite`. Theme-loader also remains directly importable via `@deckio/deck-engine/themes/theme-loader`. **Key lesson:** Never re-export Node.js-only modules from the main browser entry point. The browser/Node boundary must be enforced at the exports map level. 94 tests passing, production build verified with scaffolded project.
